# 知识卡生成器 Knowledge Card Skill

一个用于 Claude Code 的 Skill，把输入的文章、播客转文字、书摘等内容，提炼成结构化的知识卡片。

## 知识卡结构

每张卡包含六个模块：

- **【是什么】** 一句话说清本质
- **【为什么】** 说清问题根源，让读者对号入座
- **【能解决什么问题】** 从哪到哪，用具体转变描述
- **【怎么做】** 心法层 + 行动层，案例完整，细节扎实
- **【来源】** 原始内容标题、作者、相关书名/人名
- **【总结】** 原文最深刻的洞察，2-4 句话还原

## 使用方式

将 `SKILL.md` 放入 Claude Code 的 skills 目录，然后对话时说：

- 「帮我做一张知识卡」
- 「把这个整理成知识卡」
- 「@文件 帮我写知识卡」

Claude 会先从输入内容中识别所有可以做成知识卡的候选项，列出来让你选，再生成你选中的那张。

## 知识卡案例

### 案例一：20小时学会任何技能

**来源：** Josh Kaufman（《The Personal MBA》作者）播客访谈

【是什么】

20小时学会任何技能，说的是从零到"够用"只需要20小时，也就是每天40分钟坚持一个月。跟大家熟悉的"一万小时定律"不是同一件事：一万小时是追求世界级掌握，20小时是"我能用这个技能做事了"。大多数人学新东西追求的，其实是后者。

【为什么】

为什么我们觉得学新东西很难、很慢？大部分时候不是因为真的难，而是因为卡在了前10个小时。研究者把这段时间叫挫折期，技能还没建立，眼光已经有了，眼高手低最让人痛苦。成年人特别怕显得笨，自我意识一上来，学习就卡住了。更糟的是，很多人在这个阶段躲进"研究模式"，一直在看、在查、在准备，就是不动手。Josh 说，技能是在练习里建立的，不是在研究里建立的。

【能解决什么问题】

从"我很感兴趣但感觉要学很久，算了"，变成"我能在一个月内知道这个技能对不对我"。从"不知道从哪开始、研究到死也没动手"，变成有一套清晰的上手路径，在前20小时里实现从完全陌生到可以上手运转。

【怎么做】

心法层：挫折期是正常的，不是你的问题。前10小时几乎人人都很痛苦，唯一能帮你穿越这段的，是提前承诺：无论如何，我要在这件事上投入20小时。这个承诺本身就是行为心理学里的预先承诺，能显著提高你真的把事做完的概率。

行动层分五步。第一步，选一个你真的在乎的项目，不愿意预承诺20小时，说明它没你想象得重要。第二步，把目标具体化，不是"学会打碟"，是"把这首歌混入那首歌听起来顺"。第三步，把技能拆成子技能，打碟要懂BPM、调性、硬件操作，分开练，先找最关键的杠杆点。第四步，降低练习的摩擦，吉他放在随手能拿到的地方，不是锁进柜子。第五步，重量不重质，混10对不同的歌比把同一对歌磨到完美教的东西多得多。

【总结】

一万小时是为了成为世界上最好的那批人，但你一生中真正需要学的那种学习，目标从来都不是世界第一，是"我能用它做事"。20小时不是捷径，是把目标对准了正确的地方。挫折期会来，但它是结构性的，不是你的问题。预承诺20小时，就是你告诉自己：在有资格说放弃之前，我至少要先到达那里。

### 案例二：二元结果业务筛选框架

**来源：** Chris Koerner（80+ 创业经历）播客访谈

【是什么】

二元结果业务筛选框架是一套判断一门生意值不值得做的标准：输入X、输出Y，过程清晰，结果可预测，5星好评自然发生。不符合这个标准的生意，哪怕成功了，你也只是给自己建了一份工作。

【为什么】

Chris 自己开过80多家公司，踩过很多坑之后才提炼出这套框架。他开过一家家政清洁公司，找来的清洁工用各种不同的设备、总是迟到、每个客户的标准又不一样，有的客户说地板上有根头发就要投诉，有的客户根本不在意。利润薄、人不可靠、差评随时可能来。问题的根源在于它不是二元结果：你不能控制清洁工的水准，不能控制客户的预期，变量太多，每一单都在赌。

【能解决什么问题】

从"兴冲冲开始一门生意，半年后发现自己又忙又累、利润稀薄，成功了也只是给自己建了一份工作"，变成在动手之前就能判断这门生意是否值得进入。Chris 的修树公司在达拉斯做到年收入中六位数，净利润接近一半，从来没拿过低于5星的评价，不是因为团队多厉害，而是因为这门生意足够简单。

【怎么做】

用三个过滤器检查一门生意。第一，有没有二元结果，修树只有两件事：把树砍掉，或者修掉特定的枝丫，没有中间变量。第二，门槛对普通人是否可以接近，周转箱出租，租箱子送到门口，客户搬完家来取走，任何城市都可以跑通。第三，在买任何设备之前先找客户，Chris 的原则是：创业只有两件事，找客户和履行订单，找客户比履行更难，所以先做难的那件事，先在 Facebook Marketplace 挂广告，有人回应再去找怎么做。

【总结】

复杂的生意不是问题，复杂而低利润才是问题。任何生意在成功的情境下，你都要问自己：这是一门生意，还是一份工作？结果越可预测，你就越能系统化，越能交给别人做，越能真正拥有一门生意而不是被生意拥有。

## 设计原则

- 内容扎实优先于写作技巧：原文的具体数字、案例、工具必须保留，不能抽象化
- 面向读者而非自己：每张卡的目标是让第一次接触这个概念的人看懂
- 口语化，像朋友讲给你听，不是百科词条
